# Copy Rules

## Voice

Write like a precise product communicator:

- Lead with what happened or what the reader can do next.
- Prefer short paragraphs and concrete nouns.
- Use calm, direct language for transactional emails.
- Use warmth only when it helps the reader feel oriented.
- Put the business goal behind the reader's immediate need.

## Reader Psychology

Every email should answer:

1. Why am I receiving this?
2. Is anything wrong?
3. What do I need to know?
4. What should I do next?
5. What happens if I ignore this?

Do not over-explain. If an email is security-sensitive, write with trust and restraint.

## CTA Rules

- Use one primary CTA.
- Make CTA text specific: "Update payment method", "View receipt", "Reset password".
- Do not use vague CTA text like "Click here", "Learn more", or "Get started" unless the context truly supports it.
- If there is no action, do not invent a CTA. Use a status or receipt-style layout instead.

## Variable Rules

- Use variables for recipient-specific or business-specific values.
- Use examples that match the requested product and audience.
- Do not fake personalization with variables that the app is unlikely to have.
- Prefer useful variables like `first_name`, `amount`, `reference`, `dashboard_url`, `reset_url`, `invoice_number`, `support_email`.
- Use only these variable types: `string`, `number`, `boolean`, `array`, `object`.
- Treat URL, email, currency, date, ID, and reference values as `string`.
